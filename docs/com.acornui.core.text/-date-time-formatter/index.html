Properties

dateStyle

 abstract var dateStyle: DateTimeFormatStyle

The exact format of a date or time is dependent on the locale, and is platform specific, but this property will hint whether the format will be long form, medium form, or short form. The default is DateTimeFormatStyle.DEFAULT

locales

 abstract var locales: List<Locale>?

The ordered locale chain to use for formatting. If this is left null, the user's current locale will be used.

The exact format of a date or time is dependent on the locale, and is platform specific, but this property will hint whether the format will be long form, medium form, or short form. The default is DateTimeFormatStyle.DEFAULT

timeZone

 abstract var timeZone: String?

The timezone for formatting. The only values this is guaranteed to work with are "UTC" or null. Other values that will likely work based on browser or jvm implementation are the full TZ code https//en.wikipedia.org/wiki/List_of_tz_database_time_zones For example, use "America/New_York" as opposed to "EST" If this is null, the user's timezone will be used.

type

 abstract var type: DateTimeFormatType

Whether this should format the Date object as time, date, or date and time.
